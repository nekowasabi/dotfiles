# ATOK35 ローマ字スタイル設定

ATOK35 のカスタムローマ字テーブルを管理するためのファイル群。

## ファイル構成

| ファイル | 説明 |
|----------|------|
| `StUser0.plist` | ATOK35 ユーザースタイル設定ファイル（キー設定 + ローマ字テーブル） |
| `atok_convert.py` | ローマ字テキスト出力 → plist バイナリ変換コンバーター |

## コンバーター（atok_convert.py）の使い方

### 前提条件

- Python 3（標準ライブラリのみ使用、追加パッケージ不要）
- macOS + ATOK35
- ATOK「キー・ローマ字カスタマイザ」からエクスポートしたローマ字一覧テキストファイル

### 入力ファイルの準備

1. ATOK の環境設定を開く
2. 「キー・ローマ字カスタマイザ」→「ローマ字」タブ
3. 「一覧出力」でテキストファイルをエクスポート（UTF-16-BE 形式）

### スクリプトのパス設定

`atok_convert.py` 冒頭の2行を環境に合わせて変更:

```python
STYLE_FILE = Path('/Users/<username>/Library/Preferences/ATOK35/Styles/StUser0.plist')
TEXT_FILE  = Path('<ローマ字一覧テキストファイルのパス>')
```

### 実行

```bash
# バックアップを作成（推奨）
cp ~/Library/Preferences/ATOK35/Styles/StUser0.plist \
   ~/Library/Preferences/ATOK35/Styles/StUser0.plist.bak

# コンバーター実行
python3 atok_convert.py
```

出力例:
```
=== テキストファイル解析 ===
解析エントリ数: 358
=== バイナリ構築 ===
総エントリ数: 378
Section C 使用: 1878 / 6000 uint16s
Section B 使用: 378 / 1000 エントリ
生成バイナリサイズ: 16392 bytes (期待値: 16392)
=== plist 更新 ===
更新完了
```

## スタイル設定の反映方法

### 新しい Mac に反映する場合

1. ATOK35 がインストール済みであることを確認
2. ATOK のプロセスを終了（メニューバーからログアウト、またはプロセスを停止）
3. `StUser0.plist` を所定の場所にコピー:

```bash
cp StUser0.plist ~/Library/Preferences/ATOK35/Styles/StUser0.plist
```

4. ATOK を再起動（ログインし直す、または再起動）

### ローマ字設定のみ更新する場合

1. ローマ字一覧テキストファイルを用意
2. `atok_convert.py` のパス設定を確認
3. コンバーターを実行（上記「実行」セクション参照）
4. ATOK を再起動して反映

### 注意事項

- `StUser0.plist` には**キー設定**と**ローマ字テーブル**の両方が含まれる
- コンバーターは `ローマ字` キーのみを上書きし、キー設定は保持する
- 反映前に必ずバックアップを取ること
- ATOK のバージョンが異なる場合（ATOK34 以前など）、バイナリ形式が異なる可能性がある

## バイナリ形式（参考）

`ローマ字` キーは 16,392 バイト固定長の Big-Endian バイナリ:

| セクション | オフセット | サイズ | 内容 |
|-----------|-----------|--------|------|
| Header | 0 | 8 bytes | エントリ数 (uint32) + マジック 0x01000002 (uint32) |
| Section A | 8 | 384 bytes | 先頭文字ルックアップテーブル (192 × uint16) |
| Section B | 392 | 4,000 bytes | エントリテーブル (1000 × 4 bytes) |
| Section C | 4,392 | 12,000 bytes | 文字データ (6000 × uint16) |

かな文字は内部的に半角カタカナ（U+FF61〜U+FF9F）で格納され、濁音・半濁音は基底文字 + 濁点/半濁点の2文字に分解される。
